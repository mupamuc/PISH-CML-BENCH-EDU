# Подключение хранилища идей (Supabase) — 5 минут

Страница «Идеи» хранит идеи и комментарии экспертов в бесплатной облачной базе Supabase.
Пока база не подключена, страница работает в режиме «только просмотр» и показывает подсказку.

## Шаг 1. Создать проект
1. Зайдите на [supabase.com](https://supabase.com) → **Start your project** → войдите через GitHub.
2. **New project**: имя, например `pish-ideas`, пароль базы — любой (сохраните), регион — `Central EU (Frankfurt)`.
3. Подождите ~2 минуты, пока проект создастся.

## Шаг 2. Создать таблицы
1. В меню слева: **SQL Editor** → **New query**.
2. Откройте файл [`docs/supabase-setup.sql`](supabase-setup.sql), скопируйте его содержимое целиком, вставьте в редактор.
3. Нажмите **Run**. Должно появиться `Success. No rows returned`.

## Шаг 3. Скопировать два значения
1. В меню слева: **Project Settings** (шестерёнка) → **Data API**.
2. Скопируйте **Project URL** (вида `https://abcdefgh.supabase.co`).
3. Рядом, на вкладке **API Keys**: скопируйте ключ **anon / public**
   (длинная строка, начинается с `eyJ…` или `sb_publishable_…`).

> Ключ anon — публичный по замыслу: он даёт только те права, что описаны в SQL
> (читать и добавлять). Менять и удалять записи по нему нельзя, хранить его в коде безопасно.

## Шаг 4. Вставить значения в конфиг
Откройте файл `assets/collab-config.js` и замените два поля:

```js
window.COLLAB_CONFIG = {
  url: "https://ВАШ-ПРОЕКТ.supabase.co",   // ← Project URL из шага 3
  anonKey: "ВАШ-КЛЮЧ",                      // ← ключ anon / public из шага 3
  ...
};
```

## Шаг 5. Проверить
Откройте страницу «Идеи», заполните форму, отправьте. Идея должна появиться на графе.
Откройте ту же страницу во второй вкладке — новые идеи появляются там без перезагрузки.

## Управление (модерация постфактум)
Удалить спам или неудачную запись: Supabase → **Table Editor** → таблица `ideas` / `comments` →
выделить строку → **Delete row**. Комментарии и голоса удалённой идеи удаляются автоматически.
