# Спека: «Экспертный совет» — агрегатор мнений экспертов

Дата: 2026-07-10. Статус: утверждено.

## Цель
Дать экспертам возможность по одной ссылке (https://mupamuc.github.io/PISH-CML-BENCH-EDU/) предлагать идеи, критиковать и комментировать платформу ПИШ ЦИ БАС — без регистрации. Для владеющих GitHub — полноценный путь fork → PR с инструкцией на человеческом языке.

## Решения (утверждены пользователем)
- Хранилище гостевых идей/комментариев: **Supabase** (бесплатный тариф, Realtime).
- Структура графа: **идеи прикрепляются к разделам платформы** (лендинг, трекер, кабинеты, глоссарий, данные, общее); комментарии — к идеям.
- Модерация: **нет, публикация мгновенная**; удаление мусора вручную через панель Supabase.

## Состав работ

### 1. Страница `ideas.html` («Идеи»)
Три зоны:

**Граф идей (живой).** SVG + собственный force-layout (без внешних библиотек, в духе проекта «без сборки»). Узлы-якоря — разделы платформы; к ним идеи (цвет по типу: идея / критика); к идеям — комментарии (маленькие узлы). Обновление в реальном времени через Supabase Realtime (websocket) — новые идеи появляются у всех открытых вкладок без перезагрузки. Клик по идее → боковая панель: текст, автор, комментарии, кнопка «Поддержать 👍» (один голос с устройства — client_id в localStorage).

**Форма «Предложить идею».** Поля: имя (запоминается в localStorage), раздел (список), тип (идея/критика), заголовок, текст. Комментарий: имя + текст. Валидация: непустые поля, разумные лимиты длины.

**Лента «Что происходит с проектом».** GitHub REST API без авторизации (лимит 60 зап/час — достаточно): последние коммиты, форки, pull request'ы. Всё переведено на человеческий язык («Иван скопировал проект к себе», «предложил изменения — ожидают проверки»). Опрос раз в 60 сек, кэш в sessionStorage.

**Блок «Как участвовать».** Два пути: гость (представился — пишет) и GitHub (пошагово: аккаунт → Fork → правка в браузере → Pull Request, с пояснением терминов).

### 2. Supabase: схема и права
Таблицы:
- `ideas`: id uuid pk, created_at, author text, section text, kind text ('idea'|'critique'), title text, body text.
- `comments`: id uuid pk, created_at, idea_id fk, author text, body text.
- `votes`: idea_id fk + client_id text, уникальная пара (защита от повторного голоса).

RLS: anon — SELECT и INSERT; UPDATE/DELETE запрещены. Realtime включён для всех трёх таблиц. Лимиты длины полей — CHECK-констрейнты.

Конфиг фронта: `assets/collab-config.js` — URL проекта + anon key (публичный по дизайну Supabase, безопасен при включённом RLS). Если конфиг не заполнен — страница работает в режиме «только чтение GitHub-ленты» и показывает подсказку.

Настройка: SQL-файл `docs/supabase-setup.sql` + инструкция `docs/SETUP-SUPABASE.md` (5 минут: создать проект, вставить SQL, скопировать 2 значения). Альтернатива: авторизовать Supabase-коннектор Claude — тогда таблицы создаются автоматически.

### 3. Интеграция
- Ссылка «Идеи» в общей навигации (`assets/ui.js`) на всех страницах.
- README: раздел про участие экспертов.
- Стиль — существующая дизайн-система `assets/theme.css` («инженерная телеметрия», Barlow, индиго/зелёный).

## Вне охвата (осознанно)
Премодерация, ветвящиеся ответы на комментарии, редактирование/удаление своих записей, e-mail-уведомления, авторизация гостей.

## Критерии успеха
1. Гость с телефона по ссылке публикует идею за <1 минуты без регистрации.
2. Идея появляется на графе у других открытых вкладок без перезагрузки.
3. Лента показывает реальные коммиты/форки/PR репозитория на русском.
4. Эксперт без опыта GitHub по инструкции доходит до Pull Request.
5. Сайт по-прежнему открывается локально файлом и на GitHub Pages без сборки.
